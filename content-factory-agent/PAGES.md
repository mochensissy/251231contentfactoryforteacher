# 内容工厂 Agent - 页面原型说明

## 项目概述

已成功创建完整的内容工厂 Agent 页面原型，包含5个核心页面，全部基于 Next.js 15 + shadcn/ui 实现。

## 已完成的页面

### 1. 首页/仪表盘 (`/`)
**文件位置**: `app/page.tsx`

**主要功能**:
- 快速入口卡片（选题分析、内容创作、发布管理）
- 数据概览（总文章数、已发布、分析任务）
- 最近活动展示区域

**UI特点**:
- 卡片式布局，清晰的视觉层次
- 使用 Lucide 图标增强可读性
- 响应式设计，支持移动端

---

### 2. 选题分析页面 (`/topic-analysis`)
**文件位置**: `app/topic-analysis/page.tsx`

**主要功能**:
- 关键词输入和分析启动
- 实时分析进度显示（0-100%）
- 四个Tab展示分析结果：
  - 点赞TOP5：展示点赞数最高的5篇文章
  - 互动率TOP5：展示互动率最高的5篇文章
  - 高频词云：可视化展示高频关键词
  - 选题洞察：AI生成的5个选题建议，每个都有"一键创作"按钮

**交互流程**:
1. 输入关键词
2. 点击"开始分析"
3. 显示进度条（模拟分析过程）
4. 展示完整的洞察报告
5. 可保存报告或直接开始创作

---

### 3. 内容创作页面 (`/content-creation`)
**文件位置**: `app/content-creation/page.tsx`

**主要功能**:
- 双模式选题输入：
  - 从洞察报告选择（单选列表）
  - 自定义输入（标题+描述）
- 创作参数配置：
  - 文章长度（500-3000+字）
  - 写作风格（专业/轻松/故事/教程）
  - 配图数量（0-5张）
- 创作进度展示（分阶段显示）
- 文章预览（支持Markdown渲染）
- 重新生成、保存草稿等操作

**创作流程**:
1. 选择或输入选题
2. 配置参数
3. 点击"开始AI创作"
4. 显示分阶段进度（分析选题→生成大纲→创作内容→获取配图→优化排版）
5. 展示生成的文章预览
6. 保存或转到发布管理

---

### 4. 发布管理页面 (`/publish-management`)
**文件位置**: `app/publish-management/page.tsx`

**主要功能**:
- 文章列表表格展示（ID、标题、状态、平台、时间）
- 筛选功能：
  - 搜索框（按标题搜索）
  - 状态筛选（全部/草稿/待审核/已发布）
- 操作菜单（每篇文章）：
  - 预览
  - 编辑
  - 复制
  - 发布到小红书
  - 发布到公众号
  - 查看数据（已发布文章）
  - 删除
- 统计卡片（草稿/待审核/已发布数量）
- 分页功能

**状态管理**:
- `draft`（草稿）- 灰色边框徽章
- `pending_review`（待审核）- 灰色实心徽章
- `published`（已发布）- 黑色实心徽章

**平台标识**:
- 小红书 - 红色文字
- 公众号 - 绿色文字
- 多平台 - 蓝色文字

---

### 5. 设置页面 (`/settings`)
**文件位置**: `app/settings/page.tsx`

**主要配置项**:

#### Tab 1: API配置
- AI模型配置（API地址、Key、模型名称）
- 公众号文章API（地址、Key）
- Unsplash配置（Access Key）
- 连接测试功能

#### Tab 2: 平台配置
- 小红书配置（API地址、Key、账号ID）
- 公众号配置（API地址、AppID、AppSecret）
- 连接状态显示

#### Tab 3: 通用设置
- 内容创作默认参数（长度、风格、配图数）
- 选题分析默认参数（文章数量、洞察数量）
- 系统信息展示

## 技术实现要点

### 1. 组件架构
```
主布局 (app/layout.tsx)
├── 顶部导航栏 (components/main-nav.tsx)
├── 主内容区
└── shadcn/ui 组件
    ├── Card, Button, Input
    ├── Table, Badge, Tabs
    ├── Select, Progress
    └── DropdownMenu, Separator
```

### 2. 数据库设计
使用 Prisma + SQLite，包含5个主表：
- `analysis_tasks` - 分析任务
- `insight_reports` - 洞察报告
- `articles` - 文章
- `publish_records` - 发布记录
- `settings` - 系统配置

### 3. 状态管理
- 使用 React Hooks (useState)
- 模拟数据展示完整交互流程
- 所有交互都有视觉反馈

### 4. UI/UX设计原则
- 一致的间距和排版（space-y-4/6）
- 清晰的视觉层次（标题、描述、内容）
- 友好的交互反馈（hover、loading、progress）
- 完整的操作流程引导
